# Darts Game

## English

### Project Summary
The aim of this project was to demonstrate basic knowledge of the C programming language by implementing a small piece of software. We decided to create a small two-player darts game. The project was carried out by a team of four people.

### Tools Used
The project was built using the C programming language and the SDL2 library.

### Instructions
This game is designed for two players who compete against each other in a 1-on-1 match. You can start the software with the command /main in the terminal. By running /main "difficulty", you can choose a difficulty level between 1 and 15. If no difficulty level or an invalid one is provided, the game will automatically start at difficulty level 5.<br>
Afterwards, you can enter the names of both players.<br>
The game is controlled using the space bar. Pressing it allows you to set the throw’s length and height in sequence. Each player takes turns throwing three darts. The goal of the game is to reduce your score from 501 to exactly 0. The final dart must land in a double field.<br>
The point values of the dartboard correspond to the numbers placed around it. The outer ring awards double points, while the inner ring awards triple points. The circle in the center is called the bull. The outer bull/single bull is worth 25 points, and the bullseye/double bull is worth 50 points.

### Software Features
In addition to the standard gameplay, the software includes several extra features. It collects various statistics during the game, which are displayed once the match is finished. It also assists players by showing recommended “checkout paths” at the bottom of the screen. Checkout paths are combinations of fields that help players efficiently finish the game with exactly 0 points.

### My Role in the Project
Since I already had experience with darts from playing in a club, I was responsible for most of the project concept and organization. I also implemented smaller additions to the game, such as displaying checkout paths, remaining scores, and player names. In addition, I implemented the statistics system and contributed to the scoring logic.

### Project Reflection
This project was the first major software endeavor of my studies, which is noticeable in some parts of the code. Not everything is implemented optimally; some sections are inefficient, and occasional errors still occur.<br>
Nevertheless, I learned a great deal during its development, both about software implementation and about working collaboratively in a team under time pressure. Many ideas were planned, discarded, and sometimes rewritten multiple times until all components of the project fit together cohesively.

## German

### Zusammenfassung des Projektes
Innerhalb dieses Projektes sollten Grundkenntnisse im Umgang mit der Programmiersprache C gezeigt werden, indem eine kleine Software implementiert wird. Wir haben uns dazu entschieden, eine kleines Zwei-Spieler Dartsspiel zu umplementieren. Das Projekt wurde in einem Team auf vier Personen umgesetzt.

### Verwendete Werkzeuge
Verwendet wurden die Programmiersprache C und die Bibliothek SDL2 verwendet.

### Anleitung
Dieses Spiel ist für zwei Spieler*innen ausgelegt, die im 1 gegen 1 gegeneinander antreten. Man kann die Software mit dem Befehl /main im Terminal starten. Außerdem ist mit /main "Schwierigkeitsgrad" ein Schwierigkeitsgrad von 1 bis einschließlich 15 wählbar. Sollte man keinen Schwierigkeitsgrad oder einen ungültigen Schwierigkeitsgrad angeben, so wird das Spiel automatisch mit Schwierigkeitsgrad 5 gestartet.<br>
Anschließend kann man die Namen der beiden Spieler*innen eingeben. <br>
Gesteuert wird das Spiel mit der Leertaste. So können nacheinander die Länge und die Höhe des Wurfes eingestellt werden. Jede/r Spieler*in hat abwechselnd nacheinander drei Würfe. Ziel des Spiels ist es, sich von 501 Punkten auf exakt 0 Punkte herunterzuspielen. Dabei muss der letzte Pfeil in einem Doppelfeld landen. <br>
Die Felderwerte der Dartscheibe entsprechen den Zahlen, die um diese Felder herumplatziert sind. Dabei bringt der äußere Ring die doppelte Punktzahl und der innere Ring die dreifache Punktzahl ein. Der Kreis in der Mitte nennt sich Bull. Für Outer Bull bzw. Single Bull gibt es 25 Punkte. Für Bullseye bzw. Double Bull gibt es 50 Punkte.

### Funktionen der Software
Neben dem Ablauf des Spiels stellt die Software noch ein paar weitere kleine Funktionen. So sammelt sie während des Spiels verschiedene Statistiken, die nach Abschluss des Spiels angezeigt werden. Auch gibt sie den Spieler*innen eine Hilfestellung, indem untem am Bildschirm die empfohlenen "Checkoutwege" darstellt. Checkoutwege sind Felderkombinationen, mit denen man sich besonders praktisch auf exakt 0 Punkte herunterspielen kann.

### Eigene Rolle im Projekt
Da ich zu Erstellung des Projektes bereits Erfahrungen im Dartsport durch das Spielen im Verein gesammelt hatte, war ich für einen Großteil den Entwurf des Projektkonzepts und der Projektorganisation verantworltich. Außerdem habe ich kleinere Ergänzungen im Spiel implementiert, wie z.B. die Anzeige der Checkoutwege, der Restpunktzahlen und der Spielernamen, Auch habe ich die Statistiken implementiert und war an der Implementierung der Punktzahlen beteiligt.

### Reflexion des Projektes
Dieses Projekt war das erste größere Softwarevorhaben meines Studiums, was sich an einigen Stellen im Code erkennen lässt. Nicht alles ist optimal umgesetzt, manches ist ineffizient und gelegentlich treten noch Fehler auf.<br>
Dennoch habe ich während der Arbeit daran enorm viel über softwaretechnische Umsetzung sowie über die Zusammenarbeit im Team unter Zeitdruck gelernt. Viele Ideen wurden geplant, wieder verworfen und teilweise mehrfach überarbeitet, bis schließlich alle Komponenten des Projekts stimmig ineinandergreifen konnten.
